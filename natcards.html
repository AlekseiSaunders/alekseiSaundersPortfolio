Naturalist Study Cards

Helping my wife through the pandemic and learning about the iNaturalist API.

The Naturalist Study Cards project (naturaliststudycards.netlify.app) is a project I took on to:

  1. Learn to manipulate the iNaturalist API
  2. Give my wife a resource to study native animals in and around Colorado
  3. Continue to play with "card" styles, including animation

In early 2020 my wife had expressed interest in enrolling in a Colorado Naturalist program through the Audubon Society. The wait list for the program was long, and the other options to enroll didn't mesh with her work schedule.

I was looking for a fun way to help her to practice animal identification (only one part of being a naturalist) while waiting for an open slot in the course.

Technology Used

I wrote just a basic site in HTML and CSS. I used JavaScript to bring in the user inputs for region, taxa, and number of study cards.

JavaScript builds the API query url with the user inputs. I iterate through the response object to remove duplicate pictures and build an array of possible cards to build.

Based on the number of cards the user wishes to study, I randomize the images selected and build each card using the response data.

The back of the card has buttons to record whether you identified the animal correctly or not. Clicking on these buttons locks the card in place, and saves the response to a "score" that is kept in local storage.

I used Postman to help visualize the response from the iNaturalist API and to tweak parameters to get the best return data. If there aren't enough records from iNaturalist to create the number of study cards requested I alert the user and reset the page

Lessons Learned

API documentation can be a mixed bag but the iNaturalist site was very helpful in crafting the needed query url. Paired with Postman, to refine the API requests, I feel I'm getting good responses and can visualize the returned objects well.

The response object handling is complicated by my desire to filter the object for duplicate images. Moving this data into a smaller array, and then iterating over that array to build the pool of study cards challenged me. I think I've created a descent solution, but one that could be refactored for readability.

I'm looking forward to learning React. Building each card with an iteration over DOM element creation and manipulation, although working here, just looks ... ugly.

This was my first experience with keeping data in local storage. I think it serves its purpose here and it was fun to implement a score tracking system.

Site Development

I think there is a lot of potential behind the study card idea. I like the card animation and sizing, although the animation stutters if the user mouses in to the card mid-point.

Future Goals include:

  1. Add more information to card backs
  2. Refactor card creation and API response handling
  3. Use React to build the card content and html structure.
  4. Include a tally of attempts to the local storage score

Technology Used

HTML, CSS, and JavaScript. API call to the iNaturalist site to fetch images for the study cards. Local Storage to hold current and all time scores.
