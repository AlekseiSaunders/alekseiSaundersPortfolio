Movie Finder

Utilizing a Figma design file to build out a movie finder SPA

I enjoy the planning and problem solving that comes with receiving a Figma design file. The puzzle that is turning a design into a functional website that reflects the designer's intentions and maintains the desired look and feel across multiple devices and clients is a fun challenge.

The Frontend Developer path on the Scrimba learning platform is something I've been enjoying lately; Especially the solo projects that are more common as I've gotten toward the end of the course. The Movie Finder project involved using an API to fetch movies that matched a user's search criteria.

The original project called for allowing the user to save movies to a watchlist kept in local storage, allowing the watchlist to be retrieved on a separate page, and remove movies from the list, and local storage.

My rendition of the original project can be found on Netlify at Scrimba Movie Picker

I enjoyed the original, but I wanted an excuse to practice rending components with React and passing props from an API generated array into those components. My updated project (shown below) can be also be found on Netlify at Movie Finder using React

Just to be clear - in no way do I endorse Jurassic Park III

Technology Used

The final project utilizes HTML, CSS, JavaScript, and React. I use an asynchronous fetch to request and retrieve data from an API and turn the results into an array of data to use when passing props into my React component.

I utilized Postman to help visualize the data coming back from the movie database API cutting down on the trial an error sometimes needed to find usable data (not all APIs are created equally).

I use flexbox to layout both the static head elements and the dynamically generated movie components. I'm also using CSS variables to allow for the switching between light and dark modes.

Lessons Learned

I was super happy that I could use React to build out the dynamically generated cards. This was my first project where I added React to an already written code-base.

My original intent was to copy the functionality of adding to and removing from the watchlist. I managed to get the values saved to localStorage, much like the original design, but realized that I couldn't link to a second page that also built on React components.

I'm still fairly early on in my learning and working with React. I can see how passing state will help in the further development of this project and how using React router might be one option for enabling the watchlist.

Site Development

Goals are:

  • Use React to create a Header component, not just the movie card components
  • Store the addition or subtraction of a movie from the watchlist in State
  • Use that state to render the user's watchlist

Technology Used

HTML, CSS, JavaScript, React, asynchronous functions, API fetch and retrieval of data.
